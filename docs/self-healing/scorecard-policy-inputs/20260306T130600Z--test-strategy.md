# Test Strategy

## Unit Coverage
- confidence normalization and action blocking rules
- freshness parsing and staleness blocking rules
- mapping rules across runtime/integrity/incident patterns
- ranking stability and deterministic ordering
- handoff stop-condition/no-bypass construction

## Fixture-Backed Integration Coverage
Implemented in `tests/test_stage03_scorecard_policy_inputs.py`:
- degraded runtime health + high confidence => low-risk recommendation
- degraded runtime health + low confidence => evidence refresh/no-act
- strong total score + degraded dimension => targeted recommendation
- low score + severe drift => escalate-only/no-act output
- repeated incident burden => escalation recommendation
- stale scorecard => stop condition and no-act posture

## Safety Coverage
- scorecard layer cannot directly execute remediation (`no_bypass.direct_execution_allowed=false`)
- low confidence blocks action-oriented handoff
- stale evidence blocks policy evaluation handoff
- conflicting evidence suppresses policy candidates
- high-risk integrity conditions produce no-act/escalation outputs

## Regression Scope
- MCP wrapper tool registration and argument coercion
- core tool registry contract smoke for new tool exposure
- evaluate_self_healing compatibility with scorecard handoff data
