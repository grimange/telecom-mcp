## Behavior Changes

1. Outbound registration action fixed to documented contract
- Audit linkage: `DR-001`
- Change: `asterisk.pjsip_show_registration` now uses `PJSIPShowRegistrationsOutbound` and filters event-list results for the requested registration.
- Doc basis: Asterisk 22 AMI action `PJSIPShowRegistrationsOutbound`.

2. Channel detail fallback/action fixed to documented action path
- Audit linkage: `DR-002`
- Change: `asterisk.core_show_channel` and AMI fallback in `asterisk.channel_details` now use `CoreShowChannels` list action and select matching `Channel`/`Uniqueid` from `CoreShowChannel` events.
- Doc basis: Asterisk 22 AMI actions/events (`CoreShowChannels`, `CoreShowChannel`).

3. AMI command response framing hardened
- Audit linkage: `DR-003`
- Change: AMI response completion now handles `Command` flows that return `Message: Command output follows` plus deferred output chunks/`--END COMMAND--`; command output extraction now reads `Output:` lines from raw payload.
- Doc basis: Asterisk 22 AMI action `Command` semantics.

4. Empty-contact response normalization
- Audit linkage: `DR-004`
- Change: `asterisk.pjsip_show_contacts` maps `No Contacts found` to empty `items` with warning/data-quality metadata.
- Doc basis: `PJSIPShowContacts` list semantics; empty-state caveat documented explicitly.

5. Local contract docs clarified
- Audit linkage: local contract override note
- Change: documented that `asterisk.health` requires both AMI and ARI configured for current product contract posture.

## Rollback Concerns

- Medium: command framing logic is central to AMI command wrappers; rollback should preserve new connector tests.
- Low: action-name and list-action corrections are contract-safe and should not be rolled back.
